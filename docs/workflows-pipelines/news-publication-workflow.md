# Local News Article Workflow

- **Last updated:** 25 February 2026  
- **Owner:** Editorial Team  
- **Applies to:** `/local-news/` module, database-driven article system (`articles` table), SQL/HTML publication pipeline  
- **Prerequisites:** Editorial brief, access to database (phpMyAdmin), SQL insert permissions, image assets repository
- **⚠️ IMPORTANT:** All articles are now stored in the database (`articles` table) and displayed dynamically via `local-news/article.php` router. Static PHP files are deprecated.

## 1. Overview

- **Purpose:** Deliver consistent, SEO-optimised local news coverage while keeping canonical URLs, structured data, and promotions in sync.
- **Trigger:** New story assigned from editorial calendar or breaking news detected.
- **Participants:** Reporter/Writer, Editor, Web producer (publishes article), QA reviewer, Marketing (optional push plus newsletter).

## 3. Step-by-Step

1. **Editorial planning**
   - Create or update plan in `docs/content-projects/` (e.g., `PROJECT_STATUS`, article-specific plan files).
   - Capture keywords, target audience, required visuals, sources.

2. **Drafting**
   - Write content using Markdown/HTML templates (often crafted in docs first).
   - Prepare hero images (1200x630 for OG) and inline media; store under `/local-news/omr-news-images/`.

3. **Publication prep (Database-Driven System)**
   - **Primary Method: SQL Insert**
     - Create SQL file in `dev-tools/sql/` directory (e.g., `ADD-{TOPIC}-ARTICLE.sql`)
     - Insert into `articles` table with all required fields:
       - `title`, `slug`, `summary`, `content` (HTML)
       - `published_date`, `author`, `category`, `tags`
       - `image_path`, `is_featured` (1 for featured, 0 for regular)
       - `status` ('published' or 'draft')
     - Use prepared statements or proper escaping for HTML content
     - Ensure slug is unique and URL-friendly (lowercase, hyphens instead of spaces)
   - **Alternative: Admin Panel**
     - Use `/admin/news-add.php` if available (for non-technical users)
     - Form automatically handles slug generation and database insertion
   - **URL Structure:**
     - Clean URLs: `/local-news/{slug}` (via `.htaccess` rewrite)
     - Fallback: `/local-news/article.php?slug={slug}`
   - **SEO & Structured Data:**
     - Automatically generated by `local-news/article.php` router
     - Uses `core/article-seo-meta.php` for meta tags, Open Graph, Twitter Cards
     - JSON-LD structured data (NewsArticle schema, Breadcrumb schema) auto-generated
   - **⚠️ Deprecated:** Static PHP files (`/local-news/Some-Article.php`) are no longer used for new articles

4. **QA & SEO (Automatic Generation)**
   - **Meta Tags:** Automatically generated by `core/article-seo-meta.php`
     - Title, description, keywords meta tags
     - Open Graph tags (og:title, og:description, og:image, og:url)
     - Twitter Card tags
     - Canonical URL (clean URL format: `/local-news/{slug}`)
   - **Structured Data:** Automatically generated JSON-LD schemas
     - NewsArticle schema with full article metadata
     - Breadcrumb schema for navigation
   - **Manual Checks:**
     - Validate article displays correctly at `/local-news/{slug}`
     - Run article through accessibility checklist (headings, alt text for images)
     - Test share preview (Facebook/Twitter Debugger)
     - Confirm related articles module displays complementary stories
     - Verify image paths are correct (images in `/local-news/omr-news-images/`)
   - **Sitemap:** Add to sitemap (manual entry or dynamic generator) and ping Search Console if high priority

5. **Promotion & tracking**
   - Share via social channels (see marketing playbooks).
   - Include in newsletter or `docs/strategy-marketing` sequences.
   - Log article in editorial tracker (`docs/worklogs/`, `LEARNINGS.md` if major pattern emerges).

6. **Post-publication**
   - Monitor comments/feedback.
   - Update article with corrections as needed; add `updated_at` meta when substantial.
   - Archive older static files or move to `archive/` if deprecated.

## 4. Checklists

**Pre-publish**
- [ ] Editorial brief approved.
- [ ] Hero image + inline media ready.
- [ ] Structured data validated (NewsArticle + Breadcrumb).
- [ ] Links & CTA (newsletter signup, related articles) tested.

**Post-publish**
- [ ] Article live at intended URL (router + canonical match).
- [ ] Sitemap entry added/refreshed (`analytics-seo/SITEMAP-COMPLETE-LIST.md` updated if static).
- [ ] Promotion tasks completed (newsletter, social).
- [ ] Worklog entry recorded.

## 5. Edge Cases & Recovery

- **Duplicate slugs:** Router requires unique slug; append qualifier or date. Update sitemap to avoid collisions.
- **HTML in SQL:** Use `&apos;` and `&quot;` replacements or prepared statements to avoid breaking inserts.
- **Missing canonical:** If static page adds query strings, set `$canonical_url` manually before including `components/meta.php`.
- **Image licensing:** Ensure alt text and attributions included; fallback to MyOMR logo if no hero available.

## 6. Database Structure

### Articles Table Schema:
```sql
CREATE TABLE `articles` (
  `id` int(11) NOT NULL AUTO_INCREMENT,
  `title` varchar(255) NOT NULL,
  `slug` varchar(255) NOT NULL UNIQUE,
  `summary` text NOT NULL,
  `content` longtext NOT NULL,
  `published_date` datetime NOT NULL,
  `author` varchar(100) DEFAULT NULL,
  `category` varchar(100) DEFAULT NULL,
  `tags` varchar(255) DEFAULT NULL,
  `image_path` varchar(255) DEFAULT NULL,
  `is_featured` tinyint(1) DEFAULT 0,
  `status` enum('draft','published') DEFAULT 'draft',
  `created_at` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `slug` (`slug`),
  KEY `idx_status` (`status`),
  KEY `idx_published_date` (`published_date`),
  KEY `idx_category` (`category`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Key Fields:
- **slug:** Unique URL-friendly identifier (lowercase, hyphens instead of spaces)
- **content:** Full HTML article content (escape single quotes with `&apos;` in SQL)
- **status:** Must be 'published' for article to appear on homepage
- **image_path:** Relative path (e.g., `/local-news/omr-news-images/image.webp`) or absolute URL
